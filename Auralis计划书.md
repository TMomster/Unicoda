# 项目计划文档：Auralis

## Your request, our code.

---

## 1. 项目概述

**项目名称**：Auralis  
**项目类型**：桌面端 AI 编程助手  
**Slogan**：*Your request, our code.*  

Auralis 是一个**独立运行的 AI 编程伙伴**，不是 IDE 插件，也不局限在终端里。它通过调用外部大模型 API，将用户的自然语言请求转化为实际的代码操作——读文件、写代码、执行命令、搜索项目、自动修复错误。你提出需求，Auralis 以稳定、高效的节奏交付代码。

**核心定位**：纯应用层的 **Agent 编排系统**，不训练、不托管任何模型，所有智能来自外部 API（OpenAI、Claude、DeepSeek 等）。产品竞争力在于 Agent 循环设计、工具集成、上下文管理与用户体验。

---

## 2. 项目定位

- **独立桌面应用**：基于 Tauri + React，跨平台（Windows/macOS/Linux），包体小（~10MB），资源占用低。
- **面向开发者**：个人开发者、技术团队、开源贡献者，以及任何需要代码辅助的人。
- **自主 Agent**：具备规划→执行→验证→修复的循环能力，能完成多步骤、跨文件的复杂任务。
- **用户自有 API Key**：Auralis 不提供模型服务，用户配置自己的 API Key 和端点，成本透明、隐私可控。

---

## 3. 项目目标

### 3.1 核心目标（MVP）

- 提供可安装、可运行的桌面应用，用户可配置 API Key、模型端点。
- 实现 **ReAct 风格 Agent 循环**：自然语言输入 → 模型决策 → 调用本地工具 → 结果反馈 → 继续/结束。
- 支持三个基础工具：
  - 读取文件
  - 写入/创建/修改文件
  - 执行白名单内的 Shell 命令（如 `ls`, `cat`, `git status`, `npm test`）
- 清晰展示对话历史、工具调用过程、执行结果。
- 支持任务中断与取消。

### 3.2 中期目标（Beta）

- 增加 **代码检索工具**：基于 `ripgrep` 的关键词搜索 + 基于嵌入向量的语义搜索。
- **项目工作区管理**：打开任意文件夹作为项目根目录，所有操作限制在工作区内。
- **多模型切换**：支持用户自定义 API 端点和模型名称（兼容 OpenAI 格式）。
- **成本统计**：显示每次请求的 token 消耗和预估费用。
- **安全沙箱增强**：命令白名单 UI 配置、路径校验、危险操作二次确认。

### 3.3 长期目标（正式版）

- **多智能体协作**：规划器（Planner）、编码器（Coder）、测试器（Tester）分工处理复杂任务。
- **项目级长期记忆**：为每个项目独立存储向量数据库，记住项目结构、常用模式。
- **用户自定义工具**：允许用户添加自己的脚本或 API 调用作为新工具。
- **本地模型后备**：集成 Ollama，支持完全离线使用。
- **任务模板与自动化**：预设常见任务（添加单元测试、重构函数、生成文档等）一键执行。

## 4. 能力边界

### ✅ 会做

- 独立桌面应用，不依赖任何特定 IDE 或终端。
- 调用外部 LLM API（OpenAI 兼容格式）。
- 执行本地文件系统操作（读/写/创建/删除，限定在工作区内）。
- 运行预定义安全命令（用户可配置白名单）。
- 通过对话完成代码生成、修改、重构、解释、调试。
- 对大型项目进行关键词/语义检索，定位相关代码。
- 管理多个项目工作区。
- 展示每步工具调用及其结果，完全透明。

### ❌ 不会做

- 不训练、微调、托管任何大模型。
- 不提供模型 API 服务（用户需自带 Key 或使用自己的端点）。
- 不直接嵌入 VS Code、IntelliJ 等编辑器（但可提供“用外部编辑器打开”的辅助功能）。
- 不做代码补全（那是插件的领域）。
- 不替代 Git 或版本控制，但可辅助执行 Git 命令。
- 不保证绝对安全：危险命令会请求用户确认。

## 5. 技术选型

| 层级       | 技术栈                                          |
| -------- | -------------------------------------------- |
| 应用框架     | **Tauri** (Rust 后端 + Web 前端)                 |
| 前端框架     | **React 18 + TypeScript**                    |
| 样式       | Tailwind CSS                                 |
| 状态管理     | React Context + `useReducer` / Zustand       |
| 异步数据流    | TanStack React Query                         |
| Agent 循环 | 自研轻量 ReAct 引擎 + OpenAI 函数调用格式                |
| 本地文件访问   | Tauri fs 插件                                  |
| 命令执行     | Tauri shell 插件                               |
| 代码检索     | `ripgrep` 关键词搜索 + 外部嵌入 API + 本地向量数据库         |
| 向量数据库    | **LanceDB** (嵌入式，无需服务) 或 **Chroma** 嵌入式版     |
| 本地嵌入模型   | 调用外部 API（例如 OpenAI `text-embedding-3-small`） |
| 错误跟踪     | Sentry (可选)                                  |
| 更新分发     | Tauri Updater + 静态 JSON 更新清单                 |

## 6. 开发计划（里程碑）

### 阶段 0：准备与验证（1 周）

- 搭建 Tauri + React 开发环境。
- 实现简单聊天界面，能调用 OpenAI API 并显示回复。
- 验证文件读写、命令执行的权限配置。
- 确定项目名称、Logo、图标、仓库结构。

**交付物**：可运行的 demo 应用，能发送消息并得到模型回复（无工具调用）。

---

### 阶段 1：核心 Agent 循环（2 周）

- 实现 ReAct 循环：前端调用 `call_llm` 命令 → 解析 `tool_calls` → 执行本地工具 → 结果再喂给模型。
- 实现三个基础工具：`read_file`, `write_file`, `run_command`。
- 增加危险命令确认对话框（如 `rm`, `del`）。
- 实现对话历史管理及 Agent 运行状态提示。

**交付物**：用户可输入“读取当前目录下的 package.json 并解释它的依赖”等指令，Agent 能完成多步操作。

---

### 阶段 2：项目工作区与安全模型（1 周）

- 添加“打开项目”功能：用户选择一个文件夹作为工作区。
- 所有文件操作路径限制在工作区内（防止越界）。
- 实现命令白名单配置（UI 可编辑）。
- 增加危险操作的再次确认弹窗。

**交付物**：安全地管理任意项目，不会误删系统文件。

---

### 阶段 3：代码检索与上下文增强（2 周）

- 添加 `search_keyword` 工具：调用 ripgrep 在工作区内快速搜索关键词。
- 添加 `semantic_search` 工具：对外部嵌入 API 调用 + 本地向量数据库存储/查询代码块。
- 实现在对话开始时自动索引项目结构（可选，用户手动触发）。
- 在长对话中自动压缩历史或生成摘要（节省 token）。

**交付物**：用户问“找到所有使用 useState 的组件”，Agent 能定位并展示相关文件。

---

### 阶段 4：UI/UX 打磨与配置管理（1.5 周）

- 实现 API 设置面板（Key、端点、模型选择），本地加密存储。
- 增加 Token 使用量统计与预估成本显示。
- 优化对话消息展示：Markdown 渲染、代码高亮、Diff 显示。
- 增加任务取消按钮（中断 Agent 循环）。
- 实现应用内更新检查（Tauri updater）。

**交付物**：可交付给早期测试用户的 Beta 版本。

---

### 阶段 5：测试与发布准备（1 周）

- 编写自动化测试：工具执行单元测试，Agent 循环模拟测试。
- 在 Windows、macOS、Linux 上分别打包并测试。
- 编写用户文档、演示视频。
- 准备官网及下载渠道（GitHub Releases）。

**交付物**：公开发布的 MVP 版本（v0.1.0）。

## 9. 风险与应对

| 风险                 | 概率  | 影响  | 应对措施                                                        |
| ------------------ | --- | --- | ----------------------------------------------------------- |
| API 成本不可控          | 中   | 高   | 默认使用便宜的模型（如 gpt-4o-mini），提供成本上限设置，支持用户自带 Key。               |
| 模型调用格式变化           | 低   | 中   | 统一适配 OpenAI 格式，大多数厂商兼容；若不兼容，提供适配层。                          |
| 用户不愿意提供 API Key    | 高   | 高   | 预留代理方案：允许用户使用我们的代理服务（需额外付费），或支持本地 Ollama。                   |
| 命令执行安全性问题          | 中   | 高   | 严格的命令白名单 + 路径校验 + 用户确认对话框；默认禁用危险命令（如 rm）。                   |
| 大型项目上下文爆炸          | 高   | 中   | 采用 RAG 按需检索 + 对话摘要 + 限制单次读取文件大小。                            |
| 跨平台兼容性问题           | 中   | 中   | 优先支持 Windows 和 macOS，Linux 作为社区版；使用 GitHub Actions 自动化构建测试。 |
| 竞争产品（Cursor, Cody） | 高   | 中   | 走差异化路线：独立应用、不依赖 IDE、自主 Agent 长任务、本地优先。                      |

## 10. 成功指标（MVP 阶段）

- **功能完整性**：用户能通过自然语言完成至少三种常见任务（读取修改文件、运行测试、搜索代码）。
- **稳定性**：Agent 循环在 10 轮以内不出现崩溃或无限循环。
- **用户体验**：从安装到第一次成功指令不超过 5 分钟。
- **安全性**：没有用户报告的越权文件访问或危险命令误执行。
- **开源/闭源**：若开源，GitHub 星标 >100；若闭源，早期用户留存率 >40%。
